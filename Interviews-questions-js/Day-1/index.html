    Ans: An Execution Context is an abstract environment which execute javascript code.
    If code is written in global scope then it will run under global execution context.
    If the code is written un the local/functional scope then it will run under the local execution context.

Q7: Explain call(), apply() and, bind() methods. Give an example of call(), apply(), bind()

    Ans: These three are used to borrow thw function.
    call - it passes the parameter by comma separated.
    apply - it passes the parameter by by an array.
    Bind - it passes the whole function so we have to assign it to the variable and you have to call it.

    example:
    const obj1 = {
        fName : "Sam";
        lName : "johnathan"
    }

    function intro(age,place){
        console.log(`My name is ${this.fName} ${this.lName} and i am ${age} old and i am from ${place}`)
    }

    intro.call(obj1,24,India)
    intro.apply(obj1,[24,India])

    let x = intro.bind(obj1);
    x(24,India)

Q12: What is prototype chain?

    Ans: The prototype is itself an object, so the prototype will have its own prototype, 
    making what's called a prototype chain. The chain ends when we reach a prototype that 
    has null for its own prototype.

Q18: What is a Temporal Dead Zone?

    Ans: Temporal Dead Zone is the period of time during which the let and const declarations cannot be accessed.
    Temporal Dead Zone starts when the code execution enters the block which contains the let or const declaration 
    and continues until the declaration has executed.
